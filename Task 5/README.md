# Проектированиие GraphQL API

| Эндпоинт | Метод | Описание | Возвращаемые данные |
|----------|-------|----------|---------------------|
| `/clients/{id}` | GET | Получение информации о клиенте по ID | Объект `Client` (id, name, age) |
| `/clients/{id}/documents` | GET | Получение списка документов клиента | Массив `Document` |
| `/clients/{id}/relatives` | GET | Получение информации о родственниках клиента | Массив `Relative` |

### Проблемы текущей реализации

| Проблема | Описание | Влияние |
|----------|----------|---------|
| **Избыточное количество запросов** | Для получения полной информации о клиенте требуется 3 отдельных HTTP-вызова. | Высокий RPS, задержки на сети. |
| **Передача лишних данных** | REST возвращает все поля объекта, даже если клиенту нужно только `name`. | Увеличение объема трафика, замедление работы веб-приложения. |
| **Сложность расширения** | Добавление новых связей (например, транспортные средства) потребует создания нового эндпоинта и дополнительного запроса. | Рост числа вызовов, усложнение клиентской логики. |
| **Нет контроля версий** | Изменение структуры ответа может сломать существующих потребителей. | Риск обратной несовместимости. |
